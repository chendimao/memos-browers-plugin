# HTML 网页部署说明

## 目标

将当前项目打包为普通静态网页应用，部署到自己的服务器后，通过浏览器访问使用。

## 构建命令

### 仅构建 HTML/Web 版本

```bash
npm run build:web
```

### 同时构建扩展版与 Web 版

```bash
npm run build
```

构建完成后，网页版产物目录为：

```bash
dist/web
```

## 部署方式

`dist/web` 是普通静态站点目录，可以部署到任意支持静态文件托管的环境，例如：

- Nginx
- Apache
- Caddy
- 静态文件服务器
- 自己的云服务器
- Vercel / Netlify 一类静态托管平台

## 部署步骤

1. 在项目根目录执行 `npm run build:web`。
2. 将 `dist/web` 目录中的文件上传到服务器站点根目录。
3. 通过浏览器访问部署后的 URL。
4. 首次打开后，在设置页填写 Memos 的 Host 与 Token。

## 网页版保留功能

- 手动录入 memo
- 标签选择与补全
- 文件 / 图片上传
- 设置保存
- 列表查看与编辑
- 主题、语言、窗口布局相关设置

## 网页版不包含的扩展专属功能

- 网页选中文本右键注入
- 后台脚本驱动的交互
- 内容脚本注入
- 扩展图标弹窗式打开

## 部署注意事项

- 如果你的 Memos 服务与网页站点不在同一域名下，需要确保服务端允许跨域访问。
- 建议使用 HTTPS，避免浏览器对上传、认证或第三方请求做额外限制。
- 若部署在子路径下，后续可能需要根据服务器路径规则调整构建 `base` 配置。

## 验证建议

部署完成后，建议依次验证：

1. 设置保存
2. 新建 memo
3. 标签拉取与选择
4. 文件 / 图片上传
5. 列表加载与编辑

# 首次使用仅显示设置页实施计划

## 功能目标
- 当用户首次使用且 `host` 或 `token` 任一为空（含仅空格）时，插件只显示设置页面。
- 在未完成配置前，不允许进入编辑页面和列表页面。
- 配置保存成功后，恢复正常页面流转（编辑/列表/设置）。

## 技术方案
- 在 `src/App.vue` 增加统一入口守卫能力（容器层守卫），不引入新路由。
- 新增统一配置判定：`host` 与 `token` 均需 `trim()` 后非空才视为已配置。
- 用响应式守卫控制 `showSettings`：未配置时强制保持设置页打开，阻止关闭后进入编辑/列表。
- 对视图切换方法（编辑/列表切换）增加前置检查，未配置则仅打开设置页并阻断切换。

## 任务步骤
1. 在 `src/App.vue` 新增配置完整性判定计算属性（trim 后判断）。
2. 调整 `showSettings` 初始值：未配置时默认打开设置页。
3. 为未配置场景添加强制守卫（监听配置与设置面板状态，防止关闭设置页后显示编辑/列表）。
4. 在切换编辑/列表的方法中加入统一前置拦截逻辑。
5. 在模板中隐藏未配置状态下的编辑/列表切换按钮，避免误操作入口。
6. 本地构建验证：执行 `npm run build`，确认无构建错误。

## 涉及文件
- 修改：`src/App.vue`
- 文档：`md/首次使用仅显示设置页实施计划.md`

## 风险
- 若守卫逻辑与现有 `showSettings` 状态管理冲突，可能导致设置页闪烁。
- 未配置强制模式下，需避免影响“已配置用户”正常打开/关闭设置页行为。
- 现有代码较长，需确保改动最小，避免引入与上传、标签、快捷键逻辑无关的回归。

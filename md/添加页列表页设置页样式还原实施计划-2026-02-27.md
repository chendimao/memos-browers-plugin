# 添加页/列表页/设置页样式还原 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**功能目标：** 按 Figma Make 设计稿完整还原添加页、列表页、设置页的视觉与交互样式，在不改变现有业务逻辑的前提下实现统一视觉系统。

**技术方案 / 架构说明：**
先建立全局 Design Token（颜色、字号、间距、圆角、阴影、透明层级），再抽象公共控件样式（按钮、输入、卡片、标签、下拉、开关），最后逐页套用并做像素级校准。页面实现保持现有 Vue 组件结构，优先采用“样式重构 + 少量结构微调”方式降低行为回归风险。最终通过构建与手工验收确认还原质量。

**技术栈：** Vue 3 + Vite + 现有 CSS（SFC scoped/global）+ 浏览器扩展 Popup 容器

---

## 验收标准

1. 视觉还原度：三页核心区块与设计稿关键尺寸误差不超过 2px（间距、圆角、按钮高度、卡片内边距）。
2. 交互一致性：hover、focus、active、disabled 状态与设计稿视觉语义一致。
3. 一致性：跨页面公共控件（按钮、输入、下拉、标签）视觉统一，无样式漂移。
4. 稳定性：`npm run build` 通过；扩展加载后三页主流程可用（新增、查看、设置保存）。

## 潜在风险

1. 现有页面中大量 `scoped` 样式，跨组件统一 token 时可能出现优先级冲突。
2. Popup 容器宽高受设置项与浏览器环境影响，可能造成与设计稿截屏不一致。
3. 原有深色模式样式分散在多个文件，若统一 token 不完整会导致暗色回归问题。
4. 列表页内容高度、折叠逻辑与视觉容器绑定紧密，样式改动可能影响展开判断。

## 分步骤任务列表

### 任务 1：建立统一 Token 与页面骨架样式基线

**文件：**
- 修改：`src/assets/main.css`
- 修改：`src/App.vue`（仅外壳相关样式）

**步骤：**
1. 在全局样式定义颜色、圆角、阴影、透明度、间距、字体层级 token（CSS 变量）。
2. 对齐主背景渐变、顶栏、底部导航的基线视觉（包括边框、模糊、阴影层级）。
3. 将页面级重复颜色与尺寸替换为 token 引用，减少硬编码。
4. 校验浅色主题完整可用，保留现有深色入口不删除。
5. 本地构建验证：`npm run build`。

### 任务 2：统一公共控件视觉规范

**文件：**
- 修改：`src/components/CustomSelect.vue`
- 修改：`src/components/TagSelector.vue`
- 修改：`src/App.vue`（提交按钮/上传按钮/工具按钮）
- 修改：`src/views/setting.vue`（表单控件）

**步骤：**
1. 统一输入框、下拉框、标签控件的边框、圆角、阴影、焦点环和占位色。
2. 统一按钮体系（主按钮/次按钮/危险按钮）的渐变、文字、阴影与状态。
3. 统一控件尺寸阶梯（高度、内边距、图标尺寸、文本字号）。
4. 清理局部与全局冲突的旧样式，确保交互状态可预测。
5. 页面联动校验：添加页 + 列表页 + 设置页公共控件视觉一致。

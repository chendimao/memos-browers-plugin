# HTML 网页部署版本实施计划-2026-03-17

## 功能目标

在保留现有浏览器扩展打包能力的前提下，新增一个可独立部署到服务器的 HTML/Web 版本。该版本不依赖浏览器扩展 API，保留手动录入 memo、标签、文件/图片上传、设置与列表功能，并输出到 `dist/web` 目录，便于部署到任意静态服务器。

## 技术方案 / 架构说明

本次改造采用“同一套 Vue 应用核心 + 多宿主打包目标”的方式。扩展版继续使用 `popup.html` 与扩展构建脚本；网页版新增标准 `index.html` 入口，构建时输出静态站点目录 `dist/web`。运行时通过宿主标识区分 `web` 与 `extension` 两种模式，并将设置存储抽象为可在扩展存储与浏览器 `localStorage` 间切换的统一接口，避免网页版启动时依赖扩展 API。

## 潜在风险

- 现有界面逻辑较多地假设运行在扩展弹窗中，切为普通网页时可能出现初始化行为差异。
- 设置保存目前部分路径依赖扩展存储，若抽象不完整，网页版可能出现设置无法持久化的问题。
- 网页版部署后若与 Memos 服务跨域，需要用户自行在服务器侧处理 CORS 与 HTTPS 配置。
- 右键选中文本注入、后台脚本、内容脚本等扩展专属能力不会在网页版中保留。

# Safari 临时安装测试说明

## 目标

本文档用于说明如何使用本项目生成的 `dist/safari` 目录，在 macOS Safari 中进行临时安装测试。

## 构建命令

### 仅构建 Safari 产物

```bash
npm run build:safari
```

### 一次构建全部浏览器产物

```bash
npm run build
```

构建完成后，Safari 测试目录位于：

```bash
dist/safari
```

## Safari 临时安装测试步骤

1. 在项目根目录执行 `npm run build:safari`。
2. 打开 Safari。
3. 打开 Safari 的开发相关选项。
4. 按 Apple 的 Safari Web Extensions 测试方式，临时安装一个 web extension 文件夹。
5. 选择本项目生成的 `dist/safari` 目录。
6. 安装后在 Safari 中启用扩展。
7. 打开普通网页，开始手工验证功能。

## 手工验证清单

- 扩展图标点击后可以打开弹窗。
- 设置页可以保存 Host、Token、API 版本等配置。
- 在网页中选中文本后，可以通过扩展流程导入内容。
- 如果 Safari 支持右键菜单，右键添加 Memos 流程可用。
- 标签选择、文件/图片上传、列表查看等核心流程可正常使用。
- 在不同页面切换后，扩展存储的数据仍可正常读取。

## 注意事项

- 本次产物目标是“Safari 临时安装测试”，不是正式发布包。
- 如果 Safari 某个版本对 `contextMenus` 或 `action.openPopup` 的行为有限制，可能需要通过手工点击扩展图标继续测试。
- 若后续要生成正式可发布的 Safari 扩展，通常仍需要 Xcode 与 Safari Web Extension 容器工程。
